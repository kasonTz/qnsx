2. **2D性能出色**：多纹理合批方案可将DrawCall降低N倍到8倍，对微信小游戏的WebGL环境做了针对性优化
3. **社区生态最强**：国内市场占有率最高的游戏引擎之一，遇到问题容易找到解决方案，教程和插件资源丰富
4. **开发效率高**：可视化编辑器 + TypeScript + 组件化开发，适合独立开发者快速迭代
5. **包体持续优化**：V3.8.6对微信小游戏2D工程专项优化，包体减小160KB；引擎插件模式不占主包空间

#### 版本选择

- **引擎版本**：Cocos Creator **3.8.8**（当前稳定版）
- **语言**：TypeScript
- **渲染后端**：WebGL（微信小游戏默认WASM模式）

### 1.2 后端服务：Node.js + Express

#### 选择理由

1. **内容审核需求**：用户自定义文字需调用微信`msg_sec_check` API，该接口为服务端接口，必须通过后端调用
2. **技术栈统一**：前后端均使用JavaScript/TypeScript，降低独立开发者维护成本
3. **轻量级**：Node.js部署简单，云服务器2核4G即可满足初期需求
4. **微信生态兼容**：微信云开发虽然便捷，但用户选择自建服务器，拥有更高的灵活度和控制力

## 三、手势识别方案设计（重中之重）

### 3.1 技术方案：微信VKSession原生

#### 为什么选择VKSession

| 对比项 | VKSession（微信原生） | MediaPipe（Google） | 后端AI服务 |
|--------|:---:|:---:|:---:|
| 网络依赖 | ❌ 本地推理 | ❌ 本地推理 | ✅ 需要网络 |
| 延迟 | <100ms | 30-50ms | 200-500ms |
| 微信集成度 | ⭐⭐⭐⭐⭐ 官方原生 | ⭐⭐ 需手动适配 | ⭐⭐ 需自建 |
| 包体影响 | 0（系统级API） | ~2MB（WASM模型） | 0（服务端） |
| 机型兼容 | 基础库2.28.0+ | WASM兼容性不稳定 | 全机型 |
| 关键点数量 | 21个（2D）/ 16个（3D） | 33个 | 取决于模型 |

**核心优势**：VKSession是微信系统级API，不增加包体、不依赖网络、延迟最低。虽然安卓设备存在可感知延迟，但对于切水果类游戏（非毫秒级竞技），完全在可接受范围内。

#### 基础库版本要求

| 能力 | 最低基础库 | 最低微信版本 |
|------|-----------|-------------|
| VKSession基础 | 2.20.0 | — |
| Hand手势检测 | **2.28.0** | — |
| 3D关键点 | 2.28.0 | 微信 ≥ 8.1.0 |
| xr-frame稳定版 | 2.32.0（推荐） | iOS ≥ 8.0.36, 安卓 ≥ 8.0.35 |

> **建议设置最低基础库版本为2.32.0**，覆盖绝大多数活跃用户。

### 3.2 "切"动作判定算法

这是手势识别模块的核心算法，需要从21个手部关键点中识别出"挥砍/切割"动作。

#### 3.2.1 关键点选取

追踪以下关键点（基于OpenPose 21点定义）：

| 编号 | 位置 | 用途 |
|------|------|------|
| 0 | WRIST（手腕） | 手部整体位置参考 |
| 5 | INDEX_FINGER_MCP | 食指根部 |
| 8 | **INDEX_FINGER_TIP** | **食指指尖（主追踪点）** |
| 12 | MIDDLE_FINGER_TIP | 中指指尖（辅助追踪点） |

#### 3.2.2 判定流程

```
每帧获取手部关键点数据（updateAnchors事件）
    │
    ▼
步骤1：置信度过滤
    │  anchor.score < 0.5 → 丢弃该帧数据
    │
    ▼
步骤2：手指伸展检测
    │  食指指尖(8)到手腕(0)的距离 > 食指MCP(5)到手腕(0)的距离
    │  → 判定食指处于伸展状态
    │  同时检查中指指尖(12)是否也伸展（可选，提高准确率）
    │
    ▼
步骤3：速度计算
    │  记录最近3帧的食指指尖位置
    │  计算帧间位移向量 Δx, Δy
    │  计算瞬时速度 v = sqrt(Δx² + Δy²) / Δt
    │
    ▼
步骤4：速度阈值判定
    │  v > V_MIN（建议阈值：0.3 × 画布宽度/秒）
    │  → 判定为"快速移动"
    │
    ▼
步骤5：方向一致性检测
    │  最近3帧的移动方向角度偏差 < 30°
    │  → 判定为"直线挥动"（排除抖动误触发）
    │
    ▼
步骤6：连续帧确认
    │  连续2-3帧满足上述条件
    │  → 最终判定为一次"切"动作
    │
    ▼
步骤7：冷却时间
    │  触发后进入300ms冷却期
    │  防止单次挥砍被重复判定
    │
    ▼
输出：切割线段（起始点 → 当前点）
    → 传入碰撞检测系统，判断是否切中目标物
```

#### 3.2.3 关键参数

| 参数 | 建议值 | 说明 |
|------|--------|------|
| V_MIN（速度阈值） | 0.3 × 画布宽度/秒 | 低于此速度不判定为切割 |
| 确认帧数 | 2-3帧 | 连续帧满足条件才触发 |
| 方向偏差阈值 | 30° | 移动方向一致性检测 |
| 冷却时间 | 300ms | 防止重复触发 |
| 置信度阈值 | 0.5 | 低于此值丢弃数据 |
| 轨迹保留时长 | 200ms | 超过此时长的轨迹点丢弃 |

> **注意**：以上参数为初始建议值，需要在开发阶段通过大量真机测试进行调优。建议封装为可配置常量，方便迭代调整。

```
│  │         摄像头画面渲染层                  │     │
```

#### 3.3.2 集成方式选择

**推荐方案：wx.createCamera() + 帧数据绘制到Canvas**

> ⚠️ **重要平台差异**：微信小游戏（非小程序）**没有WXML/WXSS模板系统**，不支持`<camera>`组件和`cover-view`。小游戏的所有渲染都基于Canvas，需要使用JS API方式创建相机。

| 层级 | 技术 | 内容 |
|------|------|------|
| 底层 | `wx.createCamera()` + `onCameraFrame` | 摄像头帧数据获取 |
| 中层 | Canvas `drawImage` | 将帧数据绘制为游戏背景 |
| 顶层 | Cocos Creator Canvas | 游戏渲染（目标物、特效、UI） |
| 逻辑层 | VKSession（JS API） | 手势识别（不渲染画面，仅获取数据） |

**具体实现**：

1. **创建相机并获取帧数据**：
```typescript
// CameraManager.ts - 摄像头管理（仅手势模式时使用）
export class CameraManager {
  private camera: any = null;
  private frameCallback: ((frame: any) => void) | null = null;
  
  init(onFrame: (frame: any) => void) {
    this.frameCallback = onFrame;
    // 创建1x1隐藏相机（仅用于获取帧数据）
    this.camera = wx.createCamera({
      x: 0, y: 0, width: 1, height: 1,
      devicePosition: 'front',
      flash: 'off',
      success: () => console.log('相机创建成功'),
      fail: (err: any) => console.error('相机创建失败:', err)
    });
    
    // 监听实时帧数据
    this.camera.onCameraFrame((frame: any) => {
      if (this.frameCallback) {
        this.frameCallback(frame);
      }
    });
  }
  
  destroy() {
    if (this.camera) {
      this.camera.destroy();
      this.camera = null;
    }
  }
}
```

2. **将帧数据绘制到Canvas作为背景**：
```typescript
// 在游戏渲染循环中，将摄像头帧绘制为背景
// frame.data 是 ArrayBuffer 类型的图像数据
// 通过离屏Canvas中转，再绘制到主Canvas上
function renderCameraFrame(ctx: CanvasRenderingContext2D, frame: any, width: number, height: number) {
  // 创建离屏Canvas用于帧数据中转
  const offscreen = wx.createCanvas();
  offscreen.width = width;
  offscreen.height = height;
  const offCtx = offscreen.getContext('2d');
  
  // 将帧数据写入ImageData并绘制
  const imageData = offCtx.createImageData(width, height);
  const data = new Uint8ClampedArray(frame.data);
  imageData.data.set(data);
  offCtx.putImageData(imageData, 0, 0);
  
  // 绘制到主Canvas作为背景
  ctx.drawImage(offscreen, 0, 0, width, height);
}
```

> **注意**：`wx.createCamera()` 在鸿蒙OS上不支持，需要在降级方案中处理。

2. **手势识别模块**（TypeScript）：
```typescript
// GestureRecognizer.ts
export class GestureRecognizer {
  private session: any = null;
  private lastPositions: Array<{x: number, y: number, t: number}> = [];
  private cooldown: boolean = false;
  private onSlice: ((startPoint: Point, endPoint: Point) => void) | null = null;
  
  // 初始化
  init(onSliceCallback: (start: Point, end: Point) => void) {
    this.onSlice = onSliceCallback;
    this.session = wx.createVKSession({
      track: { hand: { mode: 1 } }
    });
    
    this.session.on('updateAnchors', (anchors: any[]) => {
      if (anchors.length === 0) {
        this.lastPositions = [];
        return;
      }
      
      const anchor = anchors[0];
      if (anchor.score < 0.5) return; // 置信度过滤
      
      // 获取食指指尖坐标
      const indexTip = anchor.points[8]; // landmark 8
      if (!indexTip) return;
      
      this.processFrame(indexTip.x, indexTip.y);
    });
    
    this.session.start((errno: number) => {
      if (errno) console.error('VKSession启动失败:', errno);
    });
  }
  
  // 处理每一帧
  private processFrame(x: number, y: number) {
    const now = Date.now();
    this.lastPositions.push({ x, y, t: now });
    
    // 只保留最近200ms的数据
    while (this.lastPositions.length > 0 
           && now - this.lastPositions[0].t > 200) {
      this.lastPositions.shift();
    }
    
    if (this.lastPositions.length < 3 || this.cooldown) return;
    
    // 速度计算
    const recent = this.lastPositions.slice(-3);
    const dx = recent[2].x - recent[0].x;
    const dy = recent[2].y - recent[0].y;
    const dt = (recent[2].t - recent[0].t) / 1000;
    const speed = Math.sqrt(dx * dx + dy * dy) / dt;
    
    // 速度阈值判定
    const canvasWidth = wx.getSystemInfoSync().windowWidth;
    if (speed < canvasWidth * 0.3) return;
    
    // 方向一致性检测
    // ... (计算连续帧方向角度偏差)
    
    // 触发切割
    this.cooldown = true;
    setTimeout(() => { this.cooldown = false; }, 300);
    
    if (this.onSlice) {
      this.onSlice(
        { x: recent[0].x, y: recent[0].y },
        { x: recent[2].x, y: recent[2].y }
      );
    }
  }
  
  destroy() {
    if (this.session) {
      this.session.stop();
      this.session.destroy();
    }
  }
}
```

## 四、文字自定义渲染方案设计（重中之重）

### 4.1 技术方案：离屏Canvas + 矩形碰撞 + Canvas Clip切割 + 粒子特效

#### 方案选型分析

| 方案 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| **Matter.js物理切割** | 物理效果真实 | 凹多边形分解不稳定；汉字轮廓复杂；包体+65KB | ⭐⭐ |
| **矩形碰撞 + Canvas Clip** | 零额外依赖；性能最优；实现简单 | 物理效果不如Matter.js真实 | ⭐⭐⭐⭐⭐ |
| 纯粒子系统 | 视觉效果好 | 无碰撞检测；纯视觉 | ⭐⭐ |

**推荐方案：矩形碰撞 + Canvas Clip + 粒子特效（轻量级方案）**

理由：
1. 本游戏的核心体验是"切"的爽感，而非物理模拟的真实性
2. 目标物是**文字包裹在容器中**（气泡、云朵、便签等），容器形状相对规则，矩形碰撞足够
3. 零额外依赖，不增加包体，性能最优
4. Canvas Clip可以精确实现"沿切割线分割"的视觉效果
5. 配合粒子特效，切割爽感不亚于物理方案

> **如果后续需要更真实的物理效果**，可以在V2.0版本引入Matter.js + PolyK.js方案。

#### 4.3.1 碰撞检测（矩形包围盒 + 线段相交）

```typescript
// SliceDetector.ts
export class SliceDetector {
  // 线段与矩形相交检测
  static lineIntersectsRect(
    p1: Point, p2: Point,  // 切割线段两端点
    rect: Rect               // 目标物包围盒
  ): boolean {
    // 使用Cohen-Sutherland算法或参数化方法
    // 检测线段是否与矩形的四条边相交
    return this.lineIntersectsLine(p1, p2, {x: rect.x, y: rect.y}, {x: rect.x + rect.w, y: rect.y})
        || this.lineIntersectsLine(p1, p2, {x: rect.x + rect.w, y: rect.y}, {x: rect.x + rect.w, y: rect.y + rect.h})
        || this.lineIntersectsLine(p1, p2, {x: rect.x, y: rect.y + rect.h}, {x: rect.x + rect.w, y: rect.y + rect.h})
        || this.lineIntersectsLine(p1, p2, {x: rect.x, y: rect.y}, {x: rect.x, y: rect.y + rect.h});
  }
  
  // 线段与线段相交检测
  static lineIntersectsLine(p1: Point, p2: Point, p3: Point, p4: Point): boolean {
    const d1 = this.direction(p3, p4, p1);
    const d2 = this.direction(p3, p4, p2);
    const d3 = this.direction(p1, p2, p3);
    const d4 = this.direction(p1, p2, p4);
    
    if (((d1 > 0 && d2 < 0) || (d1 < 0 && d2 > 0))
        && ((d3 > 0 && d4 < 0) || (d3 < 0 && d4 > 0))) {
      return true;
    }
    
    // 处理共线情况：端点在另一条线段上
    if (d1 === 0 && this.onSegment(p3, p4, p1)) return true;
    if (d2 === 0 && this.onSegment(p3, p4, p2)) return true;
    if (d3 === 0 && this.onSegment(p1, p2, p3)) return true;
    if (d4 === 0 && this.onSegment(p1, p2, p4)) return true;
    
    return false;
  }
  
  // 判断点pk是否在线段pi-pj上（共线前提下）
  static onSegment(pi: Point, pj: Point, pk: Point): boolean {
    return Math.min(pi.x, pj.x) <= pk.x && pk.x <= Math.max(pi.x, pj.x)
        && Math.min(pi.y, pj.y) <= pk.y && pk.y <= Math.max(pi.y, pj.y);
  }
  
  static direction(pi: Point, pj: Point, pk: Point): number {
    return (pk.x - pi.x) * (pj.y - pi.y) - (pj.x - pi.x) * (pk.y - pi.y);
  }
}
```

```typescript
export class SliceEffect {
  // 切割特效：将目标物图像沿切割线分割为两半 + 粒子
  static createSliceEffect(
    targetImage: HTMLImageElement,
    targetRect: Rect,
    sliceLine: { start: Point, end: Point },
    color: string
  ): SliceResult {
    const fragments: Fragment[] = [];
    
    // 1. 将切割线延伸到画布边界
    const extendedLine = this.extendLineToCanvas(sliceLine, targetRect);
    
    // 2. 使用Canvas Clip将图像分为两半

    const upperCanvas = wx.createCanvas();
    upperCanvas.width = targetImage.width;
    upperCanvas.height = targetImage.height;
    const upperCtx = upperCanvas.getContext('2d');
    upperCtx.save();
    upperCtx.beginPath();
    upperCtx.moveTo(extendedLine.start.x, extendedLine.start.y);
    upperCtx.lineTo(extendedLine.end.x, extendedLine.end.y);
    upperCtx.lineTo(upperCanvas.width, extendedLine.end.y);
    upperCtx.lineTo(upperCanvas.width, 0);
    upperCtx.lineTo(0, 0);
    upperCtx.lineTo(0, extendedLine.start.y);
    upperCtx.closePath();
    upperCtx.clip();
    upperCtx.drawImage(targetImage, 0, 0);
    upperCtx.restore();
    
    // 下半部分（类似）
    // ...
    
    // 3. 为每个碎片设置物理参数（初速度、旋转、重力）
    fragments.push({
      image: upperCanvas,
      x: targetRect.x + targetRect.w / 2,
      y: targetRect.y + targetRect.h / 2,
      vx: (Math.random() - 0.5) * 200,  // 水平初速度
      vy: -Math.random() * 150,           // 向上初速度
      rotation: 0,
      angularVelocity: (Math.random() - 0.5) * 10, // 旋转速度
      alpha: 1,
      gravity: 800  // 重力加速度
    });
    
    // 4. 在切割线上生成粒子
    const particles = this.createParticlesAlongLine(
      sliceLine, color, 15
    );
    
    return { fragments, particles };
  }
  
  // 沿切割线生成粒子
  static createParticlesAlongLine(
    line: { start: Point, end: Point },
    color: string, count: number
  ): Particle[] {
    const particles: Particle[] = [];
    for (let i = 0; i < count; i++) {
      const t = Math.random();
      particles.push({
        x: line.start.x + (line.end.x - line.start.x) * t,
        y: line.start.y + (line.end.y - line.start.y) * t,
        vx: (Math.random() - 0.5) * 300,
        vy: (Math.random() - 0.5) * 300,
        size: 2 + Math.random() * 4,
        color: color,
        alpha: 1,
        life: 0.5 + Math.random() * 0.5  // 粒子生命周期（秒）
      });
    }
    return particles;
  }
}
```

#### 4.3.3 碎片动画更新

```typescript
// FragmentManager.ts
export class FragmentManager {
  private fragments: Fragment[] = [];
  private particles: Particle[] = [];
  
  update(dt: number) {
    // 更新碎片
    for (let i = this.fragments.length - 1; i >= 0; i--) {
      const f = this.fragments[i];
      f.vy += f.gravity * dt;  // 重力
      f.x += f.vx * dt;
      f.y += f.vy * dt;
      f.rotation += f.angularVelocity * dt;
      f.alpha -= dt * 1.5;    // 渐隐
      
      if (f.alpha <= 0 || f.y > canvasHeight + 100) {
        this.fragments.splice(i, 1); // 回收
      }
    }
    
    // 更新粒子
    for (let i = this.particles.length - 1; i >= 0; i--) {
      const p = this.particles[i];
      p.x += p.vx * dt;
      p.y += p.vy * dt;
      p.life -= dt;
      p.alpha = Math.max(0, p.life * 2);
      p.size *= 0.98;
      
      if (p.life <= 0) {
        this.particles.splice(i, 1); // 回收
      }
    }
  }
  
  render(ctx: CanvasRenderingContext2D) {
    // 渲染碎片
    this.fragments.forEach(f => {
      ctx.save();
      ctx.globalAlpha = f.alpha;
      ctx.translate(f.x, f.y);
      ctx.rotate(f.rotation * Math.PI / 180);
      ctx.drawImage(f.image, -f.image.width / 2, -f.image.height / 2);
      ctx.restore();
    });
    
    // 渲染粒子
    this.particles.forEach(p => {
      ctx.save();
      ctx.globalAlpha = p.alpha;
      ctx.fillStyle = p.color;
      ctx.beginPath();
      ctx.arc(p.x, p.y, p.size, 0, Math.PI * 2);
      ctx.fill();
      ctx.restore();
    });
  }
}
```

```typescript
  private lastTime: number = 0;
  private elapsedTime: number = 0;

    // 注意：微信小游戏中没有全局 performance 对象，需使用 wx.getPerformance()

    // 4. 更新碎片和粒子特效

    this.updateCombo(dt);
    
    // 6. 检查游戏结束条件
    this.checkGameEnd();

    this.render();
```

```typescript
// TouchSliceInput.ts
export class TouchSliceInput {
  private isSlicing: boolean = false;
  private slicePoints: Point[] = [];
  private lastPointTime: number = 0;
  
  init() {
    // 注意：微信小游戏中没有 canvas.addEventListener，
    // 需要使用 wx.onTouchStart / wx.onTouchMove / wx.onTouchEnd
    // 注意：touch.clientX/clientY 是相对于屏幕的坐标，
    // 需要转换为相对于Canvas的坐标（减去Canvas在屏幕中的偏移量）
    const systemInfo = wx.getSystemInfoSync();
    const canvasOffset = { x: 0, y: 0 }; // Canvas在屏幕中的偏移，需根据实际布局设置

    wx.onTouchStart((e: any) => {
      this.isSlicing = true;
      const touch = e.touches[0];
      this.slicePoints = [{
        x: touch.clientX - canvasOffset.x,
        y: touch.clientY - canvasOffset.y
      }];
      this.lastPointTime = Date.now();
    });

    wx.onTouchMove((e: any) => {
      if (!this.isSlicing) return;

      const touch = e.touches[0];
      const point = {
        x: touch.clientX - canvasOffset.x,
        y: touch.clientY - canvasOffset.y
      };
      this.slicePoints.push(point);
      
      // 保留最近200ms的轨迹点
      const now = Date.now();
      while (this.slicePoints.length > 0 && now - this.slicePoints[0].t > 200) {
        this.slicePoints.shift();
      }
      
      // 速度检测
      if (this.slicePoints.length >= 2) {
        const prev = this.slicePoints[this.slicePoints.length - 2];
        const curr = this.slicePoints[this.slicePoints.length - 1];
        const dx = curr.x - prev.x;
        const dy = curr.y - prev.y;
        const speed = Math.sqrt(dx * dx + dy * dy);
        
        if (speed > MIN_SLICE_SPEED) {
          // 触发切割检测
          this.checkSliceCollision(prev, curr);
        }
      }
    });
```

### 5.4 连击系统（ComboManager）

```typescript
// ComboManager.ts
export class ComboManager {
  private comboCount: number = 0;
  private lastSliceTime: number = 0;
  private readonly COMBO_RESET_INTERVAL = 2000; // 2秒内未切中则重置

  /**
   * 切中加分物时调用
   * @returns 当前连击数
   */
  onSliceHit(): number {
    const now = Date.now();
    // 检查是否在重置间隔内
    if (now - this.lastSliceTime <= this.COMBO_RESET_INTERVAL) {
      this.comboCount++;
    } else {
      this.comboCount = 1; // 超时，重新开始计数
    }
    this.lastSliceTime = now;
    return this.comboCount;
  }

  /**
   * 切中扣分物时调用 —— 立即重置连击
   */
  onSlicePenalty(): void {
    this.comboCount = 0;
    this.lastSliceTime = 0;
  }

  /**
   * 每帧更新，检查是否超时重置
   */
  update(): void {
    if (this.comboCount > 0 && Date.now() - this.lastSliceTime > this.COMBO_RESET_INTERVAL) {
      this.comboCount = 0;
    }
  }

  /**
   * 获取当前连击数
   */
  getCombo(): number {
    return this.comboCount;
  }

  /**
   * 计算Combo加成分数
   * 规则：2连击+5分，3连击+10分，4连击+20分，5连击及以上+30分
   */
  getComboBonus(): number {
    switch (this.comboCount) {
      case 0:
      case 1: return 0;
      case 2: return 5;
      case 3: return 10;
      case 4: return 20;
      default: return 30; // 5连击及以上
    }
  }
}
```

```typescript
// SensitiveWordFilter.ts
export class SensitiveWordFilter {
  private wordSet: Set<string>;
  
  constructor() {
    // 从本地JSON文件加载敏感词（约500-1000个）
    this.wordSet = new Set(require('./sensitive-words.json'));
  }
  
  // 检查文本是否包含敏感词
  check(text: string): { safe: boolean; matched?: string } {
    for (const word of this.wordSet) {
      if (text.includes(word)) {
        return { safe: false, matched: word };
      }
    }
    return { safe: true };
  }
  
  // 模糊匹配（处理变体字、拆字等）
  checkFuzzy(text: string): { safe: boolean; matched?: string } {
    // 去除空格、特殊符号后匹配
    const normalized = text.replace(/[\s\-\_]/g, '');
    return this.check(normalized);
  }
}
```

## 八、用户系统设计

### 8.1 设计原则

> **最小化用户感知，分阶段按需获取。**

本产品的核心体验是"切文字"解压，用户信息获取不应成为进入游戏的阻碍。因此采用**分阶段策略**：

| 阶段 | 需要的用户信息 | 获取方式 | 用户感知 |
|------|--------------|---------|---------|
| **MVP** | OpenID（用户唯一标识） | `wx.login()` 静默登录 | **完全无感**，无弹窗无等待 |
| **V1.1+** | + 头像、昵称 | `wx.createUserInfoButton` 授权 | 需用户点击授权按钮 |

### 8.2 MVP方案：静默登录

#### 8.2.1 登录流程

```
用户打开小游戏
    │
    ▼
Cocos Creator 游戏启动（Loading场景）
    │
    ▼
自动调用 wx.login()（静默，无弹窗，无用户交互）
    │  获取临时登录凭证 code（有效期5分钟，一次性）
    │
    ▼
wx.request() 将 code 发送到后端 /api/auth/login
    │
    ▼
后端调用微信 code2Session 接口
    │  POST https://api.weixin.qq.com/sns/jscode2session
    │  参数：appid + appSecret + js_code=code + grant_type=authorization_code
    │  返回：openid + session_key
    │
    ▼
后端生成自定义 token（如JWT），关联 openid 存储
    │  返回 token 给前端
    │
    ▼
前端缓存 token（wx.setStorageSync）
    │  后续请求（如内容审核）携带 token
    │
    ▼
用户直接进入首页（全程无弹窗、无等待）
```

#### 8.2.2 关键设计决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| 登录时机 | 游戏启动时自动执行 | `wx.login()` 完全静默，不影响用户体验 |
| 登录失败处理 | 不阻断游戏流程，后台静默重试 | 登录仅用于获取openid，MVP阶段核心功能（游戏、自定义文字、本地存储）不依赖登录成功 |
| token存储 | `wx.setStorageSync('auth_token')` | 本地持久化，下次打开无需重新登录 |
| token刷新 | 每次启动时重新 `wx.login()` | code一次性，每次启动需重新获取 |
| session_key | 仅存后端，不下发前端 | 安全要求，session_key用于解密敏感数据 |

#### 8.2.3 前端实现

```typescript
// AuthManager.ts - 用户认证管理
export class AuthManager {
  private static TOKEN_KEY = 'auth_token';
  private static readonly LOGIN_API = 'https://your-server.com/api/auth/login';

  // 初始化：游戏启动时调用
  static async init(): Promise<void> {
    try {
      await this.login();
    } catch (e) {
      console.warn('登录失败，不影响游戏流程:', e);
      // 登录失败不阻断游戏，后续需要时再重试
    }
  }

  // 静默登录
  private static login(): Promise<void> {
    return new Promise((resolve, reject) => {
      wx.login({
        success: (res) => {
          if (!res.code) {
            reject(new Error('wx.login 未返回 code'));
            return;
          }
          // 将 code 发送到后端换取 token
          wx.request({
            url: this.LOGIN_API,
            method: 'POST',
            data: { code: res.code },
            success: (response) => {
              if (response.data && response.data.token) {
                this.saveToken(response.data.token);
                resolve();
              } else {
                reject(new Error('后端未返回 token'));
              }
            },
            fail: (err) => reject(err)
          });
        },
        fail: (err) => reject(err)
      });
    });
  }

  // 保存 token
  private static saveToken(token: string): void {
    wx.setStorageSync(this.TOKEN_KEY, token);
  }

  // 获取 token
  static getToken(): string | null {
    return wx.getStorageSync(this.TOKEN_KEY) || null;
  }

  // 判断是否已登录
  static isLoggedIn(): boolean {
    return !!this.getToken();
  }
}
```

```typescript
// 后端 - authController.ts
import axios from 'axios';

export async function login(code: string): Promise<{ token: string; is_new_user: boolean }> {
  // 1. 调用微信 code2Session 接口
  const response = await axios.get(
    `https://api.weixin.qq.com/sns/jscode2session`,
    {
      params: {
        appid: process.env.WX_APPID,
        secret: process.env.WX_APPSECRET,
        js_code: code,
        grant_type: 'authorization_code'
      }
    }
  );

  const { openid, session_key } = response.data;
  if (!openid) {
    throw new Error('code2Session 失败: ' + JSON.stringify(response.data));
  }

  // 2. 查询或创建用户记录
  let user = await db.findUserByOpenid(openid);
  const is_new_user = !user;
  if (!user) {
    user = await db.createUser({ openid, session_key });
  } else {
    await db.updateSessionKey(openid, session_key);
  }

  // 3. 生成自定义 token（如JWT）
  const token = generateJWT({ openid, user_id: user.id });

  return { token, is_new_user };
}
```

### 附录A-1：微信小游戏 vs 浏览器 API 速查

> 以下为开发过程中容易混淆的API差异，代码中必须使用微信小游戏版本。

| 功能 | 浏览器 API | 微信小游戏 API | 备注 |
|------|-----------|---------------|------|
| 创建图片 | `new Image()` / `canvas.createImage()` | `wx.createImage()` | API完全不同 |
| 触摸事件 | `canvas.addEventListener('touchstart', ...)` | `wx.onTouchStart(callback)` | 小游戏无DOM事件系统 |
| 高精度时间 | `performance.now()`（毫秒） | `wx.getPerformance().now()`（**微秒**） | 注意单位差异！ |
| 动画帧 | `requestAnimationFrame()` | `requestAnimationFrame()` | 全局支持，用法一致 |
| 全局对象 | `window` | `GameGlobal` | 小游戏没有window |
| Canvas导出 | `canvas.toDataURL()` | `canvas.toTempFilePath()`（推荐） | iOS高性能模式请优先使用 toTempFilePath |
| Canvas保存文件 | 不支持 | `canvas.toTempFilePath()` | 小游戏独有 |
| 摄像头 | `<camera>` WXML组件 | `wx.createCamera()` API | 小游戏无WXML |
| 覆盖层 | `cover-view` | **不支持** | 小游戏无DOM，UI全在Canvas上 |
